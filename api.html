class pyorient.OrientDB(host='localhost', port=2424)¶

OrientDB client object

Point of entrance to use the basic commands you can issue to the server

Parameters:
  • host – hostname of the server to connect defaults to localhost
  • port – integer port of the server defaults to 2424

Usage:

>>> from pyorient import OrientDB
>>> client = OrientDB("localhost", 2424)
>>> client.db_open('MyDatabase', 'admin', 'admin')
connect(user, password, client_id='', serialization_type='ORecordDocument2csv')¶

Connect to the server without opening any database

Parameters:
  • user – the username of the user on the server. Example: “root”
  • password – the password of the user on the server. Example: “37aed6392”
  • client_id – client’s id - can be null for clients. In clustered configurations it’s the distributed node ID as TCP host:port
  • serialization_type – the serialization format required by the client, now it can be just SERIALIZATION_DOCUMENT2CSV

Usage to open a connection as root:

>>> from pyorient import OrientDB
>>> client = OrientDB("localhost", 2424)
>>> client.connect('root', 'root')
db_count_records()¶

Returns the number of records in the currently open database.

Returns:long

Usage:

>>> from pyorient import OrientDB

>>> client.db_open('MyDatabase', 'admin', 'admin')

db_create(name, type='document', storage='plocal')¶

Creates a database in the remote OrientDB server instance.

Parameters:
  • name – the name of the database to create. Example: “MyDatabase”.
  • type – the type of the database to create. Can be either document or graph. [default: DB_TYPE_DOCUMENT]
  • storage –

    specifies the storage type of the database to create. It can be one of the supported types [default: STORAGE_TYPE_PLOCAL]:

    • STORAGE_TYPE_PLOCAL - persistent database
    • STORAGE_TYPE_MEMORY - volatile database
Returns:

None

Usage:

>>> from pyorient import OrientDB
>>> client = OrientDB("localhost", 2424)
>>> client.connect('root', 'root')
>>> client.db_create('test')
db_drop(name, type='plocal')¶

Removes a database from the OrientDB server instance. This operation returns a successful response if the database is deleted successfully. Otherwise, if the database doesn’t exist on the server, it returns an Exception

Parameters:
  • name – the name of the database to create. Example: “MyDatabase”.
  • type – the type of the database to create. Can be either document or graph. [default: DB_TYPE_DOCUMENT]
Returns:

None

db_open(db_name, user, password, db_type='document', client_id='')¶
Opens a database on the remote OrientDB Server. Returns the Session-Id to being reused for all the next calls and the list of configured clusters
Parameters:
  • db_name – database name as string. Example: “demo”
  • user – username as string
  • password – password as string
  • db_type – string, can be DB_TYPE_DOCUMENT or DB_TYPE_GRAPH
  • client_id – Can be null for clients. In clustered configuration is the distributed node
Returns:

a Information object

Usage:

>>> import pyorient
>>> orient = pyorient.OrientDB('localhost', 2424)
>>> orient.db_open('asd', 'admin', 'admin')
